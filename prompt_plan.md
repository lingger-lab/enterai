# EnterLab - 구현 계획

## 현재 상태 요약
- 핵심 기능(예약 생성/관리/알림) 구현 완료
- 랜딩 페이지 및 모바일 UI 구현 완료
- Google Cloud Build 배포 설정 완료
- 관리자 인증 (Devise) 구현 완료

## Phase 1: 안정화 및 품질 개선
- [ ] 테스트 코드 작성 (모델/컨트롤러/메일러)
- [ ] 에러 핸들링 강화 (404/500 페이지)
- [ ] 개인정보 암호화(attr_encrypted) 실제 적용 확인
- [ ] 환경 변수 누락 시 graceful 처리

## Phase 2: 관리자 기능 확장
- [ ] 예약 검색 기능 (이름/이메일/전화번호)
- [ ] 예약 일괄 상태 변경
- [ ] 관리자 계정 관리 (추가/삭제)
- [ ] 관리자 알림 설정 (이메일/SMS 선택)

## Phase 3: 사용자 경험 개선
- [ ] 예약 가능 시간 캘린더 연동
- [ ] 예약 변경/취소 기능 (사용자)
- [ ] 예약 조회 (예약번호 또는 이메일)
- [ ] 다국어 지원 (한/영)

## Phase 4: 운영 인프라
- [ ] 모니터링 및 로깅 (Sentry 등)
- [ ] 성능 최적화 (N+1 쿼리, 캐싱)
- [ ] CI/CD 파이프라인 고도화
- [ ] 백업 및 복구 전략

## 의존성
- Phase 2는 Phase 1 완료 후 진행 (테스트 기반 개발)
- Phase 3는 Phase 1 완료 후 진행 가능
- Phase 4는 독립적으로 진행 가능
