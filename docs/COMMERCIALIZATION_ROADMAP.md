## 1. 가장 먼저 결정해야 할 6가지

순서대로 결정해야 다음 결정이 가능함.

### 1.1 결제 시스템 도입 여부
가장 큰 분기점. 도입 안 하면 §2 회원 시스템 단순화 가능.

| 옵션 | 특징 | 영향 |
|---|---|---|
| **자동 결제 도입** (PG 통합) | 카드/간편결제 즉시 결제 → 정원 자동 마감 | 본격 SaaS화, 통신판매업 신고 필수 |
| **수동 입금 유지** (계좌이체) | 무료 상담 후 협의 → 입금 → 수동 확인 | 현 상태, 운영 부담 ↑, 대규모 어려움 |
| **혼합** | 무료 상담 + 결제는 상담 후 자동 발송 결제링크 | 점진적 전환 |

> **권장**: 혼합 → 점진적 자동화. 코칭은 1:1 상담 가치가 커서 무료 상담 유지 + 상담 후 결제링크 발송이 자연스러움.

### 1.2 결제 PG (도입 시)
| 옵션 | 수수료 | 세금계산서 | 정기결제 | 추천도 |
|---|---|---|---|---|
| **토스페이먼츠** | 2.8% + ₩200 | 별도 옵션 | 빌링키 ✓ | ⭐⭐⭐⭐⭐ |
| **포트원(아임포트)** | PG별 (2.7~3%) | 별도 | ✓ | ⭐⭐⭐⭐ |
| **카카오페이/네이버페이 단독** | 협상 | 부분 | △ | ⭐⭐⭐ |
| **나이스페이** | 협상 | ✓ | ✓ | ⭐⭐ B2B 무게 |

> **권장**: 토스페이먼츠. 시니어/소상공인이 카드/카카오페이 익숙. 결제링크 발송 기능 활용 가능.

### 1.3 회원 시스템 도입 여부
현재: 토큰 기반 (회원가입 없이 예약 조회/취소).

| 옵션 | 특징 | 영향 |
|---|---|---|
| **토큰 기반 유지** | 회원가입 마찰 0 | 수강 후 지속 접근 어려움 |
| **카카오 소셜 로그인** | 시니어/소상공인 친화 | 회원가입 마찰 낮음 |
| **이메일+비밀번호** | 표준 | 시니어 마찰 ↑ |

> **권장**: 결제 도입 시 카카오 소셜 로그인 도입 (시니어 친화). Devise + omniauth-kakao.

### 1.4 카카오톡 알림톡 도입 여부
현재: SMS (Naver SENS) + 이메일 (SendGrid).

| 옵션 | 비용 | 도달률 | 시니어 친화 |
|---|---|---|---|
| **SMS만 유지** | ₩20~30/건 | 95% | 평이 |
| **카카오 알림톡 추가** | ₩7~9/건 | 99% | 매우 좋음 |
| **카카오 친구톡 (마케팅)** | ₩15/건 | - | 광고용 |

> **권장**: 알림톡 추가. SMS 대비 1/3 비용 + 시니어 사용자 친숙. 카카오 비즈니스 채널 + 알림톡 인증 필요.

### 1.5 코치 수 확장
현재: 1인 코치 (Brad).

| 옵션 | 시기 |
|---|---|
| **현 상태 유지** (1명) | 월 4명 한도 — 매출 한계 ~₩600만/월 |
| **2~3명 추가** | 월 12명 가능 — ₩1,500만/월 가능 |
| **다중 코치 플랫폼화** | 코치 등록·정산·세금 처리 복잡, 별도 사업 |

> **권장**: Q1~Q2 1명 유지 → Q3에 2~3명 추가 (코치 모집 + 시스템 분리).

### 1.6 출시 타이밍 (결제 도입)
시니어 시장은 신학기·연초 효과 큼.

> **역산**:
> - 9월 결제 활성화 목표 시 → **7월 베타 결제 시작 / 6월 PG 가입 완료**
> - 결정·계약은 **5~6월 중**에 완료해야 함
> - 현 시점(5월) 기준 **여유 있음**

## 2. 카테고리별 상세

### 🟥 필수 (없으면 상용 운영 어려움)

#### §1. 결제 시스템
- PG 통합 (1.2 결정)
- 결제 흐름:
  - 무료 상담 후 → 코치가 관리자 페이지에서 "결제링크 발송" 클릭
  - 수강생이 링크 클릭 → 카드/카카오페이 결제
  - 자동 영수증 발송
- 세금계산서 자동 발급 (Bill36500 또는 PG 부가 옵션)
- 환불 처리 (현 환불 정책 그대로)
- **시간**: 2~3주
- **현 스캐폴드**: ❌ 없음. `Reservation` 모델에 결제 상태 필드 추가 필요

#### §2. 수강생 회원 시스템
- 카카오/이메일 소셜 로그인 (1.3 결정)
- 수강생 페이지:
  - 내 예약 내역
  - 결제 내역 / 영수증
  - 수강 진행 상황
  - 후기 작성
- 비밀번호 분실/재설정
- **시간**: 1~2주
- **현 스캐폴드**: 부분적 — `admin_users` (Devise) 있음, 수강생 user 모델 추가 필요

#### §3. 카카오톡 채널 + 알림톡
- 카카오 비즈니스 채널 개설
- 알림톡 템플릿 등록 (예약 확인/변경/취소/리마인더 6종)
- 사이트에 채널 추가 버튼 + 위젯
- 친구톡 마케팅 (선택)
- **시간**: 2~3일 (계정 승인 + 코드)
- **현 스캐폴드**: `KakaoNotificationJob` 구조 있음, 채널 미등록

#### §4. 영업/법적 정보 명시
- **영업 시간** 푸터/문의 페이지 명시
- **응답 시간 SLA** (예: 영업일 24h 내 답변)
- **통신판매업 신고 번호** (결제 도입 전 신고 필수)
- **부가세 표시** (사업자 분류 확정 후)
- **광고심의 책임자** (정보통신서비스 제공자 책임)
- **시간**: 1일 (정보만 확정되면)
- **현 스캐폴드**: 사업자 정보 표시 ✓, 위 항목 미반영

### 🟧 중요 (있어야 의미 있는 SaaS)

#### §5. 외부 모니터링
- **Sentry** (BE 에러 트래킹) — 무료 5K events/mo
- **HetrixTools** (다운타임 알림) — 상업용 무료 (UptimeRobot 대신)
- **알림 채널**:
  - 에러율 > 5% (5분 윈도우) → 이메일/카톡
  - 응답시간 > 2초 → 이메일
  - 사이트 다운 → 즉시 SMS/카톡
- **시간**: 1일
- **현 스캐폴드**: GA4 ✓, Sentry/HetrixTools 미통합 (사용자 가입 필요)

#### §6. 코치 다중화
현재 모델: 단일 코치 (Brad).
- `coaches` 테이블 추가 (name, bio, image, calendar_id)
- `time_slots`에 `coach_id` 추가
- 예약 시 코치 선택 또는 자동 배정
- 코치별 후기 분리
- 코치별 정산 (옵션)
- **시간**: 1~2주
- **현 스캐폴드**: ❌ 단일 코치 가정

#### §7. 수강생 학습 페이지
- 진행 중인 코칭 (주차별 진척률)
- 다음 코칭 일정
- 학습 자료 다운로드
- 카톡 커뮤니티 링크
- **시간**: 2주
- **현 스캐폴드**: ❌ (회원 시스템 후행)

#### §8. 콘텐츠/SEO
- **블로그**: AI 시대 시니어의 기회, 바이브코딩 입문, 사업 아이디어
- **사용 사례 페이지**: 실제 수강생 사례 (동의 후)
- **SEO 키워드**: "시니어 AI 코칭", "바이브코딩", "코딩 몰라도 앱 만들기"
- **네이버 블로그/카페** 보조 채널
- **시간**: 지속 작업
- **현 스캐폴드**: ❌

### 🟨 차별화 (있으면 더 잘 팔림)

#### §9. 자동화
- Sidekiq cron으로 `PurgeOldReservationsJob` 자동 실행
- 코칭 완료 시 자동 후기 요청 (이미 있음)
- 결제 미완료 알림 자동 발송
- 미완료 예약 폼 (localStorage) 자동 알림 (선택)
- **시간**: 3~5일

#### §10. 후기 자동 수집/표시
현재: 토큰 기반 후기 작성 시스템 ✓
- 후기 작성 후 자동 공개 토글 (사용자 동의 시)
- 후기 별점 + 카테고리별 필터
- 후기 통계 자동 계산 (평균 별점)
- **시간**: 1주
- **현 스캐폴드**: 후기 모델 ✓, 자동 공개 UI 부족

#### §11. 자료실
- 수강생 전용 + 일반 공개
- 전자책 다운로드 (커리큘럼 PDF)
- 코칭 영상 (선택)
- **시간**: 1주 + 콘텐츠

#### §12. 추천인 코드
- 기존 수강생이 추천한 신규 수강생에게 할인
- 추천인에게 보상 (카톡 채널 친구톡 등)
- **시간**: 1주

### 🟩 운영 체력

#### §13. 기관 영업 (B2B)
- 기업 대상 단체 코칭 패키지
- 정부/공공기관 입찰 대응
- 견적서/제안서 자동 생성
- **시간**: 별도 사업

#### §14. 정보보호 인증 (선택)
- **ISMS-P**: 정부 입찰 시 가산점
- **GIPI** (한국정보통신기술협회): 일반 사업자도 가능
- **시간**: 6개월+ (외부 컨설팅)

#### §15. API / 위탁 운영
- 외부 회계 시스템 연동
- 입금 자동 확인 (오픈뱅킹 API)
- 세무 자동화
