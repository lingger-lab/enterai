## 2. 단계 분배 평가

### 2.1 코칭 — 가격/시간 분배

| 패키지 | 가격 | 시간 | **시급** | 가격 배수 | 시간 배수 |
|--------|------|------|---------|---------|---------|
| STARTER | 980,000 | 4h | **245,000/h** | 1.00 | 1.00 |
| STANDARD | 1,480,000 | 8h | 185,000/h | 1.51 | 2.00 |
| PREMIUM | 2,490,000 | 12h | 207,500/h | 2.54 | 3.00 |

#### 🚨 발견 사항

1. **STARTER 시급 역전**: STARTER 시급(₩245k)이 STANDARD(₩185k)보다 높음
   - 의도: 단기 진입가 정책 (체험 패키지 → 업셀)
   - 부작용: "체험 패키지가 가장 비싼 시간 단가"라는 인식 가능
   - 시장 사례: 클래스101 등은 보통 진입가가 낮음 (반대 패턴)

2. **가격 vs 시간 비율 불일치**
   - 가격: 1 : 1.51 : 2.54
   - 시간: 1 : 2 : 3
   - → STANDARD/PREMIUM이 시간 대비 가성비 ↑

3. **공통 기능 "온라인 1:1 코칭 지원(30분/일, 2.5시간/주, 10시간/월)"**
   - 3개 패키지 동일 텍스트
   - STARTER (2주 코스)에 "월 10시간 지원"은 1.5개월 분량 (이상)
   - 가치 차별화 부재 — 의도 모호

### 2.2 웹 개발 의뢰 — 가격/기간 분배

| 패키지 | 가격 | 기간 (중간값) | 가격 배수 | 기간 배수 |
|--------|------|----------|---------|---------|
| BASIC | 2,900,000 | 3주 | 1.00 | 1.00 |
| STANDARD | 5,000,000 | 6주 | 1.72 | 2.00 |
| PREMIUM | 10,000,000 | 10주 | 3.45 | 3.33 |

#### 발견 사항

- ✅ 가격 vs 기간 비율 합리적 (대부분 동일 비율)
- ⚠️ **"~만원~" 표기**: 가격 상한 불명확 (시장 신뢰 ↓ 가능)
- ⚠️ **PREMIUM 스코프 모호**: "다중 AI 에이전트 대규모 개발" 10주에 가능한가?

## 3. 난이도 / 약속 평가

### 3.1 코칭 — 시니어 타깃 적합성

| 패키지 | 약속 | 4~12h에 현실적? |
|--------|------|--------------|
| STARTER | MVP 프로토타입 1개 | ⚠️ 가능하지만 매우 단순한 수준만 |
| STANDARD | 완성형 앱 + 배포 | ⚠️ 4주/8h는 빠듯, 코치 보조 多 |
| PREMIUM | 앱 + 수익화 전략 | ✅ 6주/12h면 적절 |

**우려사항**:
- "코딩 몰라도 됨" + "4시간에 MVP" = **기대치 과도 가능성**
- 시니어가 4시간 코칭만으로 자기 손으로 앱 만들기 → 현실적으로 어려움
- **실제 모델**: AI가 코드 작성 + 코치가 검수/방향 잡기 → 수강생은 기획 위주
- 사이트에 이 차이를 더 명확히 안내 권장 ("AI가 코드, 코치가 방향, 당신은 아이디어")
  - 이미 일부 카피에 반영됨, 강조 필요

### 3.2 웹 개발 의뢰 — 작업 범위 모호성

| 패키지 | 모호한 부분 |
|--------|----------|
| BASIC | "기본 UI/UX 프론트엔드 구성" — 화면 몇 개? |
| STANDARD | "결제 시스템 + 소셜 로그인" — 어떤 PG? 어떤 OAuth? |
| PREMIUM | "다중 AI 에이전트 대규모 개발" — 매우 추상적 |

→ 견적 단계에서 PRD로 명확화하지만, **사이트 표기상 모호성 ↑** = 신뢰도 영향

## 4. 법적 문제 검토

### 4.1 🔴 학원법 / 평생교육법

- **학원법 시행령**: 30시간 이상 + 학습자 10명 이상 시 학원 등록 의무
- EnterLab: 1:1 코칭, 4~12시간 → **학원 등록 불요**
- ⚠️ **개인과외교습자 신고** 검토 필요 (지자체 교육청)
- ⚠️ **출장 코칭** = 방문교습자 신고 의무 가능 (김해/장유 + 반경 100km)

### 4.2 🟡 부가세 처리 (필수 결정)

| 서비스 | 분류 | 부가세 |
|--------|------|------|
| 1:1 코칭 (교습) | **면세 가능** (교육 용역) | 0% |
| 웹 개발 의뢰 (도급) | **과세** | 10% |

→ **두 서비스 회계 분리 필요** (복합 사업자)

**현재 가격 표시 문제**:
- 코칭: "98만원" (면세면 OK, 명시는 X)
- 웹 개발: "290만원~" — ⚠️ **VAT 포함/별도 불명시** (전자상거래법 위반 가능)

### 4.3 🟡 통신판매업 신고

- 코칭 + 웹 개발 모두 "용역 제공" → 통신판매업 신고 대상 (결제 도입 시 의무)

### 4.4 🟢 표시광고법 — 후기 진정성

- 이미 "각색 시나리오" 표시 추가됨 ✓

### 4.5 🟡 사업자 업종 추가

- 코칭 + 웹 개발 = 복수 업종 → 업태/종목 추가 등록 필요할 수 있음 (세무사 확인)

### 4.6 🟢 환불 정책

- 코칭: 학원법 시행령 §18 준용 ✓
- 웹 개발: 진척률 기반 ✓
- 이미 분리 작성됨

### 4.7 🟢 분쟁 해결 / 관할 법원

- 이용약관에 명시됨 ✓

- STARTER 가격 재검토 (98만 → 88만 또는 유지)

소요: **반나절 (의사결정 多)**
영향: 패키지 구조 합리화

### 8.3 옵션 C: 종합 개편

A + B + 다음:
- 개인과외교습자 신고
- 사업자 업태 확장
- B2B 패키지 신설

소요: **수주 (외부 절차 多)**

## 10. 결론

### 10.1 한 줄 요약

> EnterLab 패키지 구조는 한국 시장에 적합하며 차별화 명확. 단, **부가세/표기/분납 정책**의 즉시 정리와 **법적 신고 검토**가 상용 운영 전 필수.

### 10.2 우선순위

1. 🔴 **부가세 + 분납 + 저작권 표시** (1~2일)
2. 🟡 **세무사 상담** (1~2주) — 면세/과세 분류, 사업자 업태
3. 🟡 **STARTER 시급 정책 재검토** (자체 결정)
4. 🟢 **개인과외교습자 신고 + B2B 패키지** (장기)

### 10.3 평가자의 추가 의견

- 현재 패키지는 **PMF(제품-시장 적합성) 확인용**으로 충분히 잘 설계됨.
- 실 수강생 5~10명 데이터 쌓이면 가격/스코프 재조정 가능.
- **무엇보다 첫 유료 수강생 확보가 우선** — 가격 정책 완벽보다 시장 검증이 더 중요.
