## 2. 한국 결제 시장 조사 (2026-05)

### 2.1 결제 패턴 4가지

#### Pattern A: 셀프 즉시 결제
- 사례: 클래스101, 야나두, 패스트캠퍼스, 콜로소
- 단가: 수만~수십만원
- 특징: 페이지 → 결제 → 영수증, 자동화 100%
- **EnterLab 적합도**: ❌ (단가/모델 부적합)

#### Pattern B: 무료 상담 → 결제 링크 ⭐
- 사례: 1:1 컨설팅, 고가 코칭, 디자인 외주, 변호사
- 단가: 100만~수천만원
- 특징: 사람 개입 후 결제 자동
- **EnterLab 적합도**: ✅ **권장**

#### Pattern C: 견적서 → 무통장 입금 (B2B)
- 사례: 기관/기업 영업
- 단가: 수백만~수억원
- 특징: 자동화 없음, 신뢰 기반
- **EnterLab 적합도**: ⚠️ B2B 출시 시 (Q4)

#### Pattern D: 정기 구독 결제
- 사례: 트레바리, 윌라, 멤버십
- 단가: 월/연 구독
- **EnterLab 적합도**: ❌ (일회성 코칭)

### 2.2 한국 결제 수단 점유율

| 수단 | 점유율 | EnterLab 적합 | 비고 |
|------|------|------------|------|
| 신용카드 | 60%+ | ⭐⭐⭐⭐⭐ | 무이자 할부 핵심 |
| 카카오페이 | 15% | ⭐⭐⭐⭐⭐ | 시니어/소상공인 친화 |
| 네이버페이 | 10% | ⭐⭐⭐⭐ | 네이버 이용자 |
| 무통장 입금 | 8% | ⭐⭐⭐⭐ | 시니어 일부 선호 |
| 토스페이 | 5% | ⭐⭐⭐ | 젊은 사용자 |
| 삼성/Apple Pay | 2% | ⭐⭐ | 모바일 |
| 휴대폰 결제 | <1% | ❌ | 100만 한도 |

### 2.3 비슷한 한국 코칭 사이트 사례

| 사이트 | 결제 방식 | 비고 |
|--------|---------|------|
| **마인드카페** | 안전 암호화 결제 (PG) | 1:1 멘탈 코칭 |
| **코아시스** | 무료 코칭 쿠폰 → 유료 전환 | 라이프 코칭 |
| **인코칭** | B2B 컨설팅 (견적) | 리더십 코칭 |
| **클래스101** | 즉시 결제 + 할부 강조 | 강의 (Pattern A) |
| **트레바리** | 정기 결제 | 독서 모임 (Pattern D) |
| **숨고/탈잉** | 매칭 후 결제 | 멘토링 플랫폼 |

## 3. PG 비교 (토스페이먼츠 vs 포트원 vs 나이스페이)

| 항목 | 토스페이먼츠 ⭐ | 포트원 (아임포트) | 나이스페이 |
|------|---------------|-----------------|-----------|
| 수수료 | 2.8% + ₩200 | PG별 2.7~3% | 협상 |
| 가입 난이도 | 쉬움 (사업자 인증) | 쉬움 | 영업 협의 |
| API 문서 | 우수 | 우수 (PG 추상화) | 보통 |
| 빌링키 (정기결제) | ✅ | ✅ | ✅ |
| 세금계산서 옵션 | ✅ 자체 + Bill36500 | ✅ Bill36500 통합 | ✅ |
| 카카오페이 | ✅ | ✅ | ✅ |
| 네이버페이 | ✅ | ✅ | ✅ |
| Apple Pay | ✅ (2024~) | ✅ | △ |
| 결제 링크 발급 | ✅ Toss 자체 페이지 | ✅ 포트원 호스팅 | △ |
| Ruby/Rails SDK | 비공식 (REST API 직접) | 비공식 | 비공식 |
| 개발자 친화도 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |
| **EnterLab 권장** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |

→ **토스페이먼츠 권장**.

### 3.1 토스페이먼츠 결제 흐름

```
1. 회사: POST /v1/payments/{paymentKey}/check (결제 생성)
2. 사용자: 결제 페이지에서 카드/카카오페이 등 선택
3. 토스: 결제 인증 (3DS 등)
4. 회사: POST /v1/payments/confirm (승인)
5. 토스 webhook → 회사 (결제 결과)
6. 회사: Payment.mark_paid!
```

## 4. EnterLab 권장 아키텍처 (Pattern B)

### 4.1 흐름

```
[1] 사이트: 사용자 → 무료 상담 신청
       ↓
[2] 코치: 카톡/전화로 1:1 상담
       ↓
[3] 합의: 패키지·일정·금액 확정
       ↓
[4] 관리자 페이지: 코치가 "결제 링크 발송" 클릭
       ├── Payment 레코드 생성 (status=issued)
       ├── pg_order_id 생성 (예: "EL-{reservation_id}-{random}")
       ├── 토스 API: 결제 페이지 URL 발급
       └── 카톡 알림톡/SMS/이메일로 링크 전송
       ↓
[5] 수강생: 링크 클릭 → 토스 결제 페이지
       ├── 카드 (무이자 할부 옵션)
       ├── 카카오페이
       ├── 네이버페이
       └── 무통장 입금
       ↓
[6] 토스: 결제 처리 → 자동 현금영수증
       ↓
[7] Webhook: 결과 회사로
       ├── Payment.mark_paid!
       ├── Reservation 결제 완료 표시
       └── 카톡 알림톡 (수강생 + 관리자)
       ↓
[8] 마이페이지: 결제 내역 표시
```

### 4.2 데이터 모델 (현 스캐폴드)

```ruby
# payments 테이블
- reservation_id
- amount (원)
- status: pending | issued | paid | failed | refunded | partially_refunded
- pg_provider: toss | kakaopay | naverpay
- pg_payment_key (PG 발급)
- pg_order_id (회사 발급)
- pg_method: 카드 | 계좌이체 | 간편결제
- issued_at, paid_at, refunded_at
- refunded_amount
- memo (관리자)
```

## 6. 무이자 할부 전략

코칭 단가 100~250만원 → 일시불 부담 ↑ → **할부가 핵심 마케팅**

| 옵션 | 회사 비용 | 사용자 부담 |
|------|---------|----------|
| **카드사 무이자 할부** (2~6개월) | 0 | 0 (카드사 부담) |
| **가맹점 무이자 할부** | 약 1~2% 추가 수수료 | 0 |
| **분납** (1차 30% + 2차 70%) | 0 | 수동 관리 |

**한국 코칭 시장 표준**: 2~6개월 카드 무이자 할부 강조. 가맹점 할부는 신중.

## 8. 부가세 (VAT) — EnterLab 특수 상황

### 8.1 면세 사업자 가능성

**학원법/평생교육법 면세 항목**:
- 학원 등록 사업자: 면세
- 1:1 개인 교습/코칭: **면세 가능성 높음** (세무사 확인 필수)

### 8.2 면세 vs 과세 결정 흐름

```
[1] 세무사 상담
   ├── EnterLab 사업자 분류 확인
   └── 면세/과세 결정
[2] 면세 사업자:
   ├── 가격에 "(면세)" 표시
   └── 세금계산서 X (계산서 가능)
[3] 과세 사업자:
   ├── 가격 표시 결정:
   │   ├── "(VAT 포함)" — 사용자 친화
   │   └── "+ VAT" — B2B 친화
   └── 세금계산서 발급 가능
```

→ **결정 필수** — 사용자 결정 사항

## 11. B2B (기관 영업) — Pattern C

### 11.1 B2B 흐름 (별도)

```
1) 기관 문의 (이메일/전화)
2) 견적서 발송 (PDF)
3) 계약서 작성/서명
4) 세금계산서 선발급 (요청 시)
5) 무통장 입금 → 수동 확인
6) 코칭 진행
7) 세금계산서 최종 (월말)
```

### 11.2 시스템 요구사항 (B2B 출시 시)

- 별도 견적 시스템 (PDF 자동 생성)
- 세금계산서 자동 (Bill36500)
- 입금 확인 (오픈뱅킹 API)
- 다중 결제 (분납/계약금)

→ Q4 이후 도입 검토

## 12. 분할 결제 / 분납

| 옵션 | 처리 |
|------|------|
| **카드 무이자 할부** (1~12개월) | PG 자동 처리 |
| **2회 분납** (1차 + 2차) | Payment 레코드 2개 생성, 별도 결제 링크 |
| **계약금 + 잔금** | Payment.status='partially_paid' 활용 |

→ 초기엔 카드 무이자 할부만으로 충분. 분납은 B2B에서.

## 16. 보안 / 컴플라이언스

### 16.1 PCI-DSS

- 카드 정보 직접 저장 X
- 토스가 토큰화 처리
- 회사는 pg_payment_key (참조 ID)만 저장

### 16.2 PIPA (결제 관련)

- 결제 금액·일시 보존 5년 (전자상거래법)
- 결제 정보 암호화 (현 PII와 동일)
- 처리방침에 결제 관련 보유 기간 명시 (이미 작성됨)

### 16.3 결제 사기 대응

- IP 기반 rate limit (rack-attack)
- 동일 카드 반복 결제 차단 (토스 자체 기능)
- 의심 거래 알림 (선택)
